# Сканер Библиотеки Liconism.com

Часть 1. Приложение скачивает и анализирует библиотеку PDF-документов Международного научного интернет-проекта
"Лингвистический Иконизм" и выполняет поиск книг, связанных с темой иконизма в восточных языках
(китайский, японский, корейский). 

Часть 2. На полученных данных (один из многих вариантов), проводится дообучения синтаксического парсинга с целью построения синтаксического дерева частей речи в предложении и определения их морфологии. Ссылка на вторую часть проекта https://github.com/e9krivo/--1.git

> Путём изменения списка ключевых слов приложение может быть перенастроено под любую другую тему поиска.

## Структура проекта

```
.
├── downloader.py        # скачивает все книги библиотеки в папку downloads/
├── scanner.py     # сканирует PDF и ранжирует книги по релевантности
├── keywords.json        # список ключевых слов с весами
└── downloads/           # создаётся автоматически при запуске downloader.py
```

## Метод: TF-IDF

**TF (Term Frequency)** -- как часто термин встречается в конкретной книге. Используется сглаженная
версия `log(1 + tf)`, чтобы книга с 200 упоминаниями не доминировала над книгой с 20 упоминаниями в 10 раз.

**IDF (Inverse Document Frequency)** -- насколько термин редок по всей коллекции: `log(N / df)`, где `N` -- общее число
документов, `df` -- в скольких из них встречается термин. Слово есть в каждой книге → IDF ≈ 0. Слово только в двух
книгах из 600 → высокий IDF.

**TF-IDF = TF × IDF** -- высокое значение только тогда, когда термин одновременно частый в данной книге и редкий в
остальных.

Скоры всех документов нормируются в диапазон `[0, 1]`, где `1.0` -- самая релевантная книга в коллекции.

## Список ключевых слов

Задаётся в файле `keywords.json`. Ключевые слова трактуются как **префиксы**: `"ономатоп"` найдёт `"ономатопея"`,
`"ономатопоэтический"` и любые другие формы. Это позволяет не перечислять все словоформы вручную.

Формат файла:

```json
{
  "ключевое_слово": вес,
  "keyword": weight
}
```

Вес (`weight`) -- целое число, определяющее относительную важность термина при суммировании TF-IDF компонент.
Рекомендуемые значения: `50` для тематических терминов, `1–10` для вспомогательных.

## Установка и запуск

Установка зависимостей:

```bash
pip install requests beautifulsoup4 urllib3 pymupdf scikit-learn numpy tqdm
```

Скачивание библиотеки (~600 книг, занимает продолжительное время):

```bash
python downloader.py
```

Сканирование и ранжирование:

```bash
python scanner.py
```

## Конфигурация scanner.py

Все параметры находятся в начале файла:

| Параметр        | По умолчанию      | Описание                                                          |
|-----------------|-------------------|-------------------------------------------------------------------|
| `pdf_folder`    | `"downloads"`     | Папка с PDF файлами                                               |
| `keywords_file` | `"keywords.json"` | Файл ключевых слов                                                |
| `cutoff`        | `0.01`            | Минимальный нормированный балл `[0..1]`. `0` -- вывести все книги |
| `top_n`         | `0`               | Вывести только top-N результатов. `0` -- без ограничений          |

## Вывод программы

Программа выводит:

1. **Статистику совпадений** -- для каждого ключевого слова показывает, какие конкретные термины из TF-IDF словаря были
   найдены и сколько их. Помогает отлаживать `keywords.json`.

2. **Список книг** -- отсортированный по убыванию релевантности, с визуальным индикатором и численным скором:
